RLS enabled on all core tables, every policy gated on company_id = current_company_id().

- The same per-company RLS pattern repeats in ~30 migrations (directory, parcels, carriers, lockers, assets, inventory, routes, templates…).

10. Authentication (NIS2)

- web/src/routes/_app.tsx (session guard), login.tsx (generic error — no user enumeration), welcome.tsx (invite/set-password flow), web/src/lib/password.ts (CSPRNG password generation).

- supabase/config.toml — refresh-token rotation on, rate limits set, MFA (TOTP) present but disabled (needs Supabase Pro).

Gaps the sweep surfaced (worth a follow-up list)

1. No MFA/SSO yet — TOTP disabled in config.toml, minimum_password_length = 6, no Entra ID. Known roadmap item, but it's the biggest NIS2 gap.

2. No retention/purge policy — no cron job or config for expiring audit_log, parcel_events, import_runs, inbound_files, or parcel-photos. Append-only forever is defensible for audit, but GDPR expects a stated retention period,
and photos of parcels have no lifecycle at all.

3. gateway/.env is world-readable (644) on disk — the hook secret and SFTPGo admin password are readable by any local user on that box.

4. Webhook auth is a static ?token= in the URL, not an HMAC-signed body — no replay protection, and URL tokens can leak via proxies/logs. Applies to both the SFTP and email hooks.

5. Anonymization covers the employees row, but free-text fields elsewhere (parcel notes, sender names) could still contain names — no sweep exists for those.
